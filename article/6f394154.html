评论组件的选择

经过一个上午的寻找比对，以及未来省一点注册账号的麻烦，我选择了GitHub带的gitment评论组件。gitment的实现原理是把GitHub的issues直接拉过来当评论组件，作为全世界最大的同性交友网站，我相信有GitHub账号的人还是有很多的。

开始

第一步：准备

先到GitHub上注册一个应用，地址如下：

地址 然后点左边菜单栏最下面的的Developer settings

这里要注意的点是，那几个空到底填些啥：

Application name ：填写这个应用的名字（填啥无所谓），比如gitment。

Homepage URL ：这个填写你博客的网址，在浏览器地址栏里复制黏贴就行了。

Application description :顾名思义，这个是描述，填啥无所谓，比如博客评论系统

Authorization callback URL: 这个比较关键，我一开始也以为自己死在上面了（其实不是）。这里要填写也是你博客的地址（未免之后产生各种问题，建议到浏览器地址栏复制）。这个东西好像是一个跳回链接，gitment是一个第三方评论插件嘛（大概），所以当用户通过博客下面的log in跳转到GitHub授权的之后他们不是还要在回到博客地址这里来嘛，所以这个地址就是在GitHub那边授权完毕之后跳转到的地址。

注册完之后你就会得到两个很关键的东西client ID和client secret 记好它们，接下来很有用。

然后：

到GitHub上再注册一个repo用来放评论，就和一开始给博客创建repo时一样

第二步

到本地themes\kilia\_config.yml文件中配置相关信息，我用的是yilia主题，你当然可能用的是其他主题，但我想都到了装评论系统的地步了，找到themes文件夹还是没有问题的。

一般正常的主题设计者都会在themes\kilia\_config.yml中奖评论相关的各种插件放在一块的，像gitment、多说、畅言之类的，你只要找到相关的，把它前面的#去掉（#表示注释），或者把它后面的指从false改成true就行。

然后就是具体的配置：

gitment_owner: #你的 GitHub ID

gitment_repo: ‘ ‘ #存储评论的 repo

gitment_oauth:

client_id: ‘ ‘ #client ID

client_secret: ‘ ‘ #client secret

然后hexo g hexo d 就好了，千万不要hexo s不要在本地测试博客系统

然后就可以看到文章下面有评论系统了。
